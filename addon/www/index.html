CalDav Integration für HomeMatic - hm_caldav
Allgemeines

Dieses CCU-Addon liest eine ics Datei von der angegebenen URL. In der Konfiguration kann angegeben werden, welche Termine darin über welche Systemvariable in der CCU bekannt gemacht werden. Läuft ein konfigurierter Termin so wird dies im Status der Systemvariablen dargestellt.

Unterstützte CCU Platformen
  • HomeMatic CCU3 / RaspberryMatic
  • HomeMatic CCU2
  • HomeMatic CCU1
Installation als CCU-Addon
  1. Download des aktuellen Addon-Releases von Github
  2. Installation des Addon-Paketes (hm_caldav-X.X.tar.gz) mittels WebUI Oberfläche der CCU
  3. Konfiguration des Addons über WebUI zugüngliche Konfigurationswebseite
Manueller CUxD SystemExec Aufruf

Zusätzlich zum automatischen, intervallbasierten Aufrufen von hm_caldav ist es auch möglich mittels CUxD-basierter SystemExec Aufrufe (www.cuxd.de) in der CCU-basierten Programmierumgebung eine Ausführung von hm_caldav zu bestimmten Zeiten durchführen zu lassen. Hierzu kann man in HomeMatic-Programmen die folgende beispielhafte Anweisung hinzufügen:

dom.GetObject("CUxD.CUX2801001:1.CMD_EXEC").State("/usr/local/addons/hm_caldav/run.sh <iterations> <waittime> &");

Hierbei gilt zu beachten, dass bei <iterations> und <waittime> jeweils die Anzahl der Iterationen und eine gewünschte Wartezeit zwischen den Iterationen hinzugefügt werden kann. Ein Beispiel eines solchen Aufrufen kann sein:

dom.GetObject("CUxD.CUX2801001:1.CMD_EXEC").State("/usr/local/addons/hm_caldav/run.sh 5 2 &");

Dies wird bei Ausführung dieser Anweisung dann hm_caldav 5-mal hintereinander aufrufen mit jeweils 2 Sekunden Pause zwischen den Aufrufen.
